//클러스터상태

}

모든 데이터 출력하기
GET /.kibana/_search
{
  "query":{
    "match_all": {}
  }

CRUD 관련

데이터삽입
#제품삽입
POST /products/_doc
{
  "name": "Snack",
  "price": 5000,
}

언더스코어(_)_는 사용자 변수가 아닌 API 명임을 나타냅니다. doc에 데이터를 넣습니다. id 값이 명시되지 않으면 elastic search에서 생성합니다.

GET
GET /products/_doc/100

id가 100번인 데이터를 가져옵니다.

Update
POST /products/_update/100
{
	"doc":{ "price": 5000 }
}

아이디가 100번인 데이터의 price 값을 500으로 변경합니다.

Script
POST /products/_update/100
{
  "script": {
    "source": "ctx._source.price--"
  }
}

id가 100번인 데이터의 price를 1만큼 낮추어 저장합니다. ctx는 document 자신을 _source는 품고 있는 데이터값을 의미하며 100번으로 따로 주지 않으면 전체를 업데이트할 수 있습니다._ ###### Script + 변수사용

"source": "ctx._source.price -= params.amount",
"params": { "amount" : 1000}

변수와 함께 증감할 수 있습니다.

삭제
DELETE /products/_doc?100

100번 데이터를 삭제합니다.
