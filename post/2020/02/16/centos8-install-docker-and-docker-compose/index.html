Centos8에서 Docker와 Docker Compose 설치

2020년 02월 16일 Harry The Great

Centos 기존에 아래와같이 도커를 설치했을경우 에러가 발생했습니다.

#버전호환문제로 error가 발생
yum install -y yum-utils device-mapper-persistent-data lvm2
yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
yum install -y docker-ce

Docker는 엔터프라이즈 버전과 커뮤니티 버전(CE)가 존재하며 커뮤니티 버전의 경우 cantos에서의 버전지원문제가 있다고 합니다.

## 저장소추가
dnf config-manager --add-repo=https://download.docker.com/linux/centos/docker-ce.repo
## 저장소확인
dnf list docker-ce #Docker CE Stable이 보여야합니다.
## 설치하기
dnf install docker-ce --nobest -y

dnf명령어란? RPM 기반 리눅스 배포판을 위한 패키지 관리도구입니다. 만약 dnf --version 커맨드로 아무것도 출력되지 않는다면 yum install dnf 커맨드로 설치합니다.

# 서비스 시작
service docker start

# 재시동시 자동으로 시작되도록 등록
systemctl enable docker

## 도커 버전 확인
docker -v

## 도커구동 간단한 테스트
docker run hello-world

테스트 수행 후 "Hello From Docker!"가 출력된다면 정상적으로 설치되었습니다.

# Docker Compose 설치

curl -L "https://github.com/docker/compose/releases/download/1.25.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

만약 curl이 설치되어있지 않다면 dnf install curl -y 명령어를 먼저 수행합니다.

#실행권한부여
chmod +x /usr/local/bin/docker-compose
#버전확인
docker-compose --version
